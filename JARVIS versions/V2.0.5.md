# JARVIS Version 2.0.5 Manifest

## Target: Autonomous Reminder & Wellness Engine (Section 13)
**Status:** ACTIVE

### Core Enhancements:

1. **Scheduled Time Reminders (Section 13a):**
   - Fire spoken reminders at any user-specified clock time (e.g. `12:30`).
   - Support for **daily repeating** and **one-time** reminder modes.
   - One-time reminders auto-cancel after firing via `schedule.CancelJob`.
   - Time input parsed in `HH:MM` (24-hour) format with graceful error handling.

2. **Interval Wellness Reminders (Section 13b):**
   - **Hydration alert** — every 2 hours, active window 09:00–21:00.
   - **Focus break alert** — every 25 minutes, active window 09:00–17:00.
   - Window enforcement handled inside callbacks — no job cancellation overhead.
   - Both reminders speak personalised JARVIS-toned messages.

3. **Volume Control Integration (Section 13c):**
   - Uses `pycaw` (`AudioDevice.EndpointVolume` API) to raise master volume to 80% before speaking if below 50%, then restores the original level.
   - Non-destructive — original volume always preserved.

4. **Isolated TTS Channel (Section 13d):**
   - Reminder speech routed through `pyttsx3` (Windows SAPI5).
   - Completely separate audio channel from the primary edge-tts/pygame pipeline — zero contention.
   - Speech dispatched via a thread-safe `queue.Queue` worker.

### New Voice Commands:
| Command | Effect |
|---|---|
| `remind me daily at HH:MM <message>` | Add a daily repeating reminder |
| `remind me at HH:MM <message>` | Add a one-time reminder |
| `list reminders` | Speak all active user-defined reminders |
| `cancel all reminders` | Clear all user-defined reminders (built-ins preserved) |

### New Files:
- `core/reminder_engine.py` — Self-contained `ReminderEngine` class (270 lines)

### Modified Files:
- `main.py` — Fault-isolated import, `__init__`, `boot()`, `shutdown()`, `CommandRouter.route()`
- `requirements.txt` — Added `schedule`, `pyttsx3`, `pycaw`, `comtypes`

### Notes:
This version introduces autonomous time-awareness to JARVIS — the system now proactively monitors Sir's wellbeing and schedule without requiring any manual invocation. The reminder engine runs in isolated daemon threads alongside the existing async command loop, ensuring zero disruption to core voice recognition and response pipelines.
